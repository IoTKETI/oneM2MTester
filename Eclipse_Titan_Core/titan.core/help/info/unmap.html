unmap

The operation is used to terminate the mapping between the ports of a test component to the ports of the test system interface - it is the opposite operation of map. .

  • The operation has effect only if the mapping  to be removed has been created beforehand.

Related keywords:

  • create
  • mtc
  • self
  • system

unmap (component_referenceA:port_referenceA , component_referenceB:port_referenceB);  

  • The unmap keyword introduces the termination of the port mapping. The indices A and B refer to the two sides of the mapping.

  • component_reference is one of the following:

    • a reference returned by the create operation,
    • the keyword self when referring to the component where unmap is given,
    • the keyword system when referring to the test system interface,
    • the keyword mtc when referring to the main test component.
       

  • port_reference points out the port that should be released from the mapping.

Example:

unmap(vc_KompRef:Port11_PCO, system:Port2_PCO);

Port11_PCO of the component with the reference vc_KompRef  is disconnected from the port Port2_PCO of the test system interface.
