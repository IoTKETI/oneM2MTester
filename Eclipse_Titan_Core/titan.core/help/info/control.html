control

The module control part may contain local definitions and describes the execution order (possibly repetitive) of the actual test cases.

Related keywords:

  • execute
  • with

control {  [ local_definitions ] control_statements }  [ with attributes ]

  • The control keyword introduces the module control part.

  • the optional local_definitions part may contain variable, constant and timer definitions.

  • the control_statements state the execution order of the test cases (cf. the keyword execute).

  • attributes: see the keyword with for details.

Example

control { var boolean vl_mistni; 
   execute( tc_pad1()); 
   execute( tc_pad2());
}

The module Principal declared here has an integer module parameter (tsp_vypinac), an enumerated type (Examplg) defined in the module definitions part. The control part contains the execution of the test case tc_prov.
